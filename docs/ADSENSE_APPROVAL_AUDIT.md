## 2. 신뢰 페이지 점검

### 2.1 존재 여부

| 페이지 | 존재 | 파일/위치 |
|--------|------|-----------|
| **About (소개)** | ❌ 없음 | 전용 about.html 없음. index 홈 섹션·app.html #siteIntro에 분산된 소개만 존재 |
| **Privacy Policy** | ✅ 있음 | privacy.html |
| **Contact** | ✅ 있음 | contact.html |

### 2.2 품질 점검

**Privacy Policy (privacy.html)**  
- **분량:** 본문 약 2,000자 이상 (정보성 문서로 충분).  
- **구조:** 시행일, 수집 정보, 이용 목적, 쿠키/AdSense, 제3자 제공, 보유 기간, 이용자 권리, 아동, 문의 등 8개 절.  
- **외부 링크:** Google adssettings, aboutads.info, policies.google.com/technologies/ads 만 사용 → 스팸 아님.  
- **톤:** 문서형, 정책 문구 적절.  
- **판단:** 최신 유지만 하면 됨. **유지.**

**Contact (contact.html)**  
- **분량:** 본문 약 450~500자 수준 → **800자 미만.**  
- **내용:** 이메일, Gmail/메일앱 링크, 문의 시 기기/브라우저 등 안내.  
- **외부 링크:** mailto, Gmail 작성 링크만 → 스팸 아님.  
- **판단:** **보강 권장.** 800자 이상 정보성 문단 추가(응답 정책, 운영 시간, 개인정보 문의 안내 등).

**About (소개 페이지)**  
- 전용 페이지 없음.  
- index.html 홈 섹션: 기능 소개(달력·루틴·타이머·메모·위젯)로 일정 분량 있으나 “서비스 소개·운영 주체·목적”이 한 페이지에 정리된 About 문서는 아님.  
- **판단:** **About 전용 페이지 추가 권장** (800자 이상, 문서형 톤).

## 3. 얇은 페이지(Thin Content) 탐지

전체 HTML 기준 **실질 본문(보이는 텍스트)** 기준 추정.

| 페이지 | 추정 본문 길이 | 실질 콘텐츠 | 판단 |
|--------|----------------|-------------|------|
| **index.html** | 1,200자 이상 | 홈 헤더 + 기능 5섹션(설명문) | 유지 |
| **app.html** | 800자 이상 | #siteIntro 다단락 소개 + 푸터 | 유지 |
| **privacy.html** | 2,000자 이상 | 정책 8절 | 유지 |
| **contact.html** | 450~500자 | 문의 안내 + 이메일 | **보강** |
| **insight.html** | 350~400자 | 제목 1줄 + 3섹션(빈 상태 또는 제목 목록만) | **보강** |
| **checklist.html** | 1,500자 이상 | 체크리스트 항목·안내문 | **유지(비공개 권장)** |
| **widget.html** | 50자 미만 | 제목만, 본문은 JS 동적 | **기능 전용** – Thin이지만 위젯 목적상 유지 |
| **navera8d96...html** | 1줄 | 검증용 텍스트 | **유지** (검증 전용) |

### 3.1 페이지별 권장 조치

- **contact.html:** 800자 이상으로 보강(아래 4. 수정 초안 참고).  
- **insight.html:**  
  - “등록된 게시물이 없습니다”만 반복되면 Thin으로 보일 수 있음.  
  - 카테고리별 1~2문단 설명(Brain/Mind/Action 소개), 또는 인사이트 페이지 목적·활용법을 400자 이상 추가 권장.  
- **checklist.html:**  
  - 개발자용이라 일반 사용자 콘텐츠는 아님.  
  - **sitemap/메인 네비에 없음** → 검색 노출 최소화 권장. 필요 시 robots.txt에서 `/checklist.html` Disallow 검토.  
- **widget.html:**  
  - 기능성 페이지. 콘텐츠 확장보다는 sitemap 제외 유지가 일반적.  
  - 현재 sitemap에 없음 → **유지.**

## 4. 내부 링크 구조 점검

### 4.1 도달 가능성 (2~3클릭)

- **index** → privacy, contact, insight: 1클릭 (푸터/사이드바). ✅  
- **app** → privacy, contact, insight: 1클릭. ✅  
- **insight** → index, privacy, contact: 1클릭. ✅  
- **privacy** → index, contact: 1클릭. ✅  
- **contact** → index, privacy: 1클릭. ✅  

**문제:**  
- **index ↔ app** 간 상호 링크 없음.  
  - index에서 “앱(달력 바로 쓰기)” 진입점 없음.  
  - app에서 “랜딩/홈”으로 돌아가는 링크 없음(홈 버튼은 in-app 전환).  
- **About 전용 페이지**가 없어 “소개”로 갈 수 있는 진입점 없음.

### 4.2 Dead link

- 푸터·사이드바의 `privacy.html`, `contact.html`, `insight.html`, `index.html` 모두 동일 도메인 상대 경로.  
- 해당 파일 존재 → **404 위험 낮음.**

### 4.3 네비게이션·레이아웃

- index / app: 사이드바 + 푸터 구조 동일.  
- insight: 사이드바 구조 유사, “인사이트”만 현재 페이지 표시.  
- privacy, contact: 단일 카드 레이아웃으로 일관됨.

**권장:**  
- index 푸터 또는 홈 섹션에 “앱 사용하기” → `app.html` 링크 추가 검토.  
- app 푸터 또는 상단에 “홈으로” → `index.html` 링크 추가 검토.  
- About 페이지 추가 시, 푸터에 “소개” 링크 추가.

## 5. 광고 코드 점검

- **현재:** HTML 주석에 “승인 전용(광고 완전 제거 버전)” 명시.  
- **검색 결과:**  
  - app.js 내 `reloadAdsense()` 호출부만 존재(기능 변경 금지로 미수정).  
  - `adsbygoogle`, `googlesyndication` 등 실제 광고 스크립트/인라인 코드는 HTML에 없음.  
- **ads.txt:** `google.com, pub-5786516862887733, DIRECT, ...` 존재.

**결론:**  
- **콘텐츠 최상단 즉시 광고:** 해당 없음(광고 미삽입).  
- **버튼/클릭 유도 근처 광고:** 해당 없음.  
- **광고 밀도·모바일 과다:** 해당 없음.  

승인 후 광고 삽입 시 참고:  
- 콘텐츠(본문) 시작 후 일정 스크롤 또는 첫 문단 아래에 배치.  
- CTA/주요 버튼 바로 위·아래에는 배치하지 않기.

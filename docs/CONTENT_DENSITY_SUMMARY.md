## 1. 페이지별 추가 문단 요약

### contact.html
- **분량:** 본문 약 1,100자 수준 (900자 이상 충족).
- **추가 내용:**
  - **문의 유형:** 일반 문의, 오류 제보, 제휴 문의, 개인정보 관련 문의를 구체화하고 각각 제목 표기([일반문의], [오류제보], [제휴문의], [개인정보]) 안내.
  - **응답 정책:** 이메일 접수만 사용, 회원가입 불필요, 영업일 기준 1~3일 내 확인·회신, 보안·개인정보 문의는 우선 확인, 영업·광고 회신 없음 명시.
  - **개인정보처리방침 연결:** 개인정보 문의는 [개인정보] 표기 및 privacy.html 링크 문장 추가.
- **기타:** h2 스타일, ul 스타일 추가. 메일 본문 기본 템플릿에 문의 유형 선택 안내 반영.

### index.html
- **이미지 설명 2~3개 추가:**
  - **달력 이미지 아래:** 월별 그리드에서 날짜 클릭 시 일정·할 일 등록·체크, 상단 이달의 목표로 매일 확인 가능하다는 1문장.
  - **루틴 이미지 아래:** 매일·매주 반복 항목 등록, 주간 뷰에서 체크, 색상·이모티콘으로 진행 상황 확인 가능하다는 1문장.
  - **메모 이미지 아래:** 날짜별 메모 저장, 목록에서 선택해 위젯으로 띄우기, PC 한쪽에서 메모 보며 작업 가능하다는 1문장.
- **하단 문단 (500~700자):**
  - 새 섹션 "Jay Calendar 소개" 추가 (border-top으로 구분).
  - **사이트 목적:** 웹에서 개인 일정·할 일·루틴·메모를 한 화면에서 관리, 월별 달력 기준 등록·체크, 주간 루틴, 날짜별 메모·위젯 제공.
  - **사용자에게 도움되는 점:** 설치 없이 브라우저만 사용, 데이터는 기기 저장소, 메뉴·위젯으로 기능 선택 사용, 이달의 목표 상단 노출, 날짜별 구분으로 파악 용이.
  - **기능 소개 요약:** 월간 달력, 일정·할 일 등록·체크, 주간 루틴, 날짜별 메모·위젯, 타이머 등; 상세 사용법은 메뉴 진입 후 화면 안내 참고.

### insight.html
- **J캘린더 활용 포스팅 20개 (정적 HTML):**
  - 새 섹션 "J캘린더 활용 포스팅" 추가. 각 글은 H3 제목 + H4 소제목 3~4개 + 문단으로 구성, 문제 해결·설명형, 광고/감성 문구 없음.
  - **주제:** ① 웹 달력 월별 일정 관리 ② 이달의 목표 상단 활용 ③ 할 일·일정 구분 등록 ④ 루틴 주간 설정·체크 ⑤ 메모 날짜별 정리 ⑥ 위젯으로 달력만 띄우기 ⑦ 여러 타이머 동시 사용 ⑧ 날짜 클릭 후 일정 추가 절차 ⑨ 반복 루틴 설정 유의점 ⑩ 메모만 위젯으로 열기 ⑪ 브라우저 저장소 저장 방식 ⑫ 일정 색상·이모지 ⑬ 주간 루틴 체크리스트 확인 ⑭ 달력에서 오늘 날짜 찾기 ⑮ 할 일 기한(시작·종료일) ⑯ 메모 제목·내용 작성 요령 ⑰ 위젯 창 크기·닫기 ⑱ 저장 데이터 초기화 시 주의 ⑲ PC·모바일 계정 없이 사용 ⑳ 이달의 목표 수정·삭제.
  - **형식:** `.insight-article` 단위, H3(제목)·H4(소제목)·p(본문), 글당 800자 이상.
  - **스타일:** `.insight-article` margin/padding, h3/h4/p 폰트 크기·간격만 추가 (레이아웃 구조 변경 없음).
